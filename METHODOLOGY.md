# 会议纪要 → 道法术器 · 高密度拆解 SOP

> 把 50-90 分钟的现场分享，拆成「读完就能上手做事」的可复用结构化笔记。
>
> 灵感来源：生财有术《拆解-01-王煜全》黄金模板 + 狗哥商业模式三层框架完整稿。

---

## 一、底层信念（不能忘）

1. **拆解 ≠ 内容缩写** —— 是把讲者的"暗知识"翻译成"任何人都能套用的 SOP"
2. **信息密度 > 篇幅** —— 同样 150 行，要塞下 8 道 / 5 法 / 12 数据 / 15 金句
3. **必须有可验证的"真东西"** —— 真实结果、真实数字、真实故事，缺一不可
4. **道法术器是漏斗，不是装饰** —— 道决定走多远 / 法决定怎么走 / 术决定动手做什么 / 器是趁手工具

---

## 二、源稿读取协议（最容易出错的环节）

❌ **不要做的**：只读「智能总结」/「录音总结」段。这是 AI 已经压缩过的结论。

✅ **必须做的**（按顺序）：

| 顺序 | 读什么 | 找什么 |
| --- | --- | --- |
| 1 | **课堂实录 / 章节概要** | 时间戳里的具体故事、对话、案例 |
| 2 | **金句精选** | 逐字摘录原话，准备做"金句"段 |
| 3 | **重点速览 / 考点重点** | 讲者认为最重要的判断 |
| 4 | **核心概念** | 抽取认知层的"独立词条" |
| 5 | **录音总结** | 最后用来验证自己有没有漏掉大块 |

> 黄金标准：每一篇拆解要至少读完源稿 **100% 的字数**，不是只看摘要。

---

## 三、拆解结构模板（8 个固定段）

每段都有"密度基线"——达不到就不交付。

### 段 1：嘉宾身份（200-400 字）

不只是"谁"，是"凭什么这场分享值得听"。包含：

- 个人履历曲线（早期 → 转折点 → 现在）
- 战绩表（用表格列 3-5 个身份/产品/数字）
- 一句"自我定义"的原话引用
- 反差点（最戳人的那个对比）

### 段 2：真实结果（5-8 项）

不是泛泛的"做得好"，是**具体数字 + 时间锚点**。每项一行：
- "X 个月做到 Y"
- "Z 板块每月稳定 N 万"
- "扶持的合作方做到 W"

### 段 3：关键数据（KPI 网格 8-15 项）

把每个数字变成可独立引用的"信息原子"。格式：

| 数据 | 含义 |
| --- | --- |
| 6,000 单/天 | 别人帮卖货时 X 在睡觉 |
| 70 万 | 上月新盘子利润 |
| ... | ... |

### 段 4：核心故事（2-3 个鲜活场景）

每个故事要有：
- 角色（讲者本人或其他具名人物）
- 冲突（之前 vs 之后 / 困境 vs 转折）
- **直接引用的原话**（用 blockquote）
- 一句"这个故事戳人的地方"的元解释

### 段 5：方法框架（道法术器，最厚的一段）

#### 道（5-8 个底层认知）
每个 1 个子标题 + 4-5 句解释 + 1 个具体例子。**不能只列要点。**

#### 法（3-5 个方法论）
每个含：
- 名字（一句话讲清逻辑）
- 子结构（表格 / 编号步骤）
- 至少 1 个完整案例

#### 术（4-7 个具体动作）
不是抽象建议，是"今天 / 明天 / 这周就能做的事"。

#### 器（5-8 个工具）
含成本（如有）、用途、适用场景。

### 段 6：工具清单（汇总表）

把 5 段里所有工具做一张总表，方便用户直接照单买/装。

### 段 7：金句（12-18 条）

**逐字原话，不改写**。用 blockquote 形式排版。
分类标注（战略 / 执行 / 心法 / 反共识）会更好。

### 段 8：给你的最大启示（3-5 条 actionable）

每条结构：**"如果你正在做 X，那就 Y"**。
不要写正确但抽象的话（"要努力"），要写"今天就该问自己的具体问题"。

## 四、密度自检清单（交付前必过）

- [ ] 总行数 ≥ 150
- [ ] 嘉宾身份 ≥ 200 字 + 1 张战绩表 + 1 句自我定义引用
- [ ] 道 ≥ 5 个，每个有子标题
- [ ] 法 ≥ 3 个，每个有表格或编号步骤
- [ ] 关键数据 ≥ 8 项
- [ ] 核心故事 ≥ 2 个，至少 1 个含直接引用
- [ ] 金句 ≥ 12 条
- [ ] 表格使用 ≥ 4 处
- [ ] blockquote 引用 ≥ 3 处
- [ ] 给你的启示 ≥ 3 条 actionable

---

## 五、避坑清单

| 反面教材 | 正面做法 |
| --- | --- |
| 用"AI 时代"开头每段 | 删掉所有空泛大词，直接讲具体 |
| 列要点不展开 | 每个要点必须有子句解释 + 例子 |
| 写"做得好"不给数据 | 找到具体数字、时间、规模 |
| 把金句改写成自己的话 | 严格逐字摘录原话 |
| 道、法、术、器混在一起 | 严守边界：道是认知，术是动作 |
| 故事讲成"他做了…又做了…" | 用对话、引用、转折点重塑叙事 |
| 一上来就写"今天就能做" | 等到段 8 再给行动指令 |

## 七、什么时候不该用这套 SOP

- 源稿不足 30 分钟 / 不到 5,000 字 → 强行套会注水
- 讲者只在讲个人经历没讲方法论 → 改成"故事 + 反思"格式
- 是对谈 / 圆桌 / 答疑 → 用 Q&A 结构 + 关键洞察分类，不强套道法术器
