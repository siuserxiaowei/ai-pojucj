## 四、核心故事

最戳人的是**两款工具的真实对比**：

| 维度 | OpenClaw | 爱马仕 |
| --- | --- | --- |
| 稳定性 | 每次更新易出 bug | 自带稳定性更强 |
| 学习成本 | 用户需自行安装技能 | 大量预设技能，无需从零搭建 |
| 进化能力 | 依赖人工 | 具备自我进化能力，自动沉淀技能 |
| 适用场景 | 高度定制化 | 快速上手 + 日常运营 |

第二个戳人的是**Reddit 养号的细节**：

> "在 Reddit 平台做日常养号，设定每天仅跑 2 次任务，分为早晚各一次。AI 会筛选高质量帖子发布评论，避免平台检测，目前运行稳定。"

意思是：**多 Agent 不是越多越好——节奏控制比频率更重要。**

## 五、方法框架（道法术器）

### 道（底层认知）

- **原来 AI 是单点聊天工具，现在要拆分角色组成团队**
- **必须设置大总管角色**——用户只跟大总管沟通
- **类似公司管理层只对接中层**——这是组织铁律
- **稳定性 > 灵活性**——Agent 不稳定就是负资产

### 法（方法论）

#### 1. 多 Agent 团队架构原则

- **大总管 + 部门主管 + 执行 Agent**（三层）
- 用户不直接和执行 Agent 沟通
- 减少繁琐细节，提高决策效率

#### 2. 网站全流程 10 步拆分

```
找需求 → 判断关键词 → 搜索意图
  → 写定位 → PRD → 设计 → 开发
    → 上线 → 测试验收 → 运营启动 → 数据复盘
```

- 95% 由 AI 完成
- 5% 用户决策保留
- 关键是"打通工具链 + 做好决策"

#### 3. 工具选型对比

- 高度定制化业务 → OpenClaw
- 快速上手 + 日常运营 → 爱马仕
- 出海业务 → 必须云端部署

### 术（具体策略）

#### 出海自动化运营方案

- **数据采集**：第三方 MCP 实现 X 平台热点每日采集
- **养号**：Reddit 早晚各 1 次，避免平台检测
- **内容**：AI 筛选高质量帖子发评论
- **复盘**：采集数据回流系统迭代

#### 云端部署 vs 本地部署

| 维度 | 本地 | 云端（腾讯云海外节点） |
| --- | --- | --- |
| 成本 | 较高（卖个 mini） | 可控 |
| IP 静态 | 不稳定 | 静态稳定 |
| 出海要求 | 无法满足 | 完美匹配 |
| 链路 | 直连 | NC 加密 |
| 实测 | 经常掉线 | 丝滑稳定 |

### 器（工具）

- **OpenClaw**：定制化 Agent 框架
- **爱马仕**：稳定 Agent 框架（含预设技能）
- **腾讯云海外节点**：出海部署
- **NC 加密链路**：远程操控
- **第三方 MCP**：数据采集
- **Reddit / X 平台**：出海获客阵地

## 八、给你的最大启示

如果你在做出海或多 Agent 业务：

1. **设你的"大总管"**：不要让用户直接和执行 Agent 对话
2. **把你的业务拆成 10 步以上**：找出哪些 95% 能给 AI、哪些 5% 必须自己做
3. **如果出海，立刻上云端**：本地部署在 IP 稳定性上完全不够用
